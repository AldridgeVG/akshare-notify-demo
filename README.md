# akshare-notify-demo

基于 [AKShare](https://github.com/akfamily/akshare) 的 ETF 多指标实时监控工具，结合 **MACD + KDJ + 成交量** 给出综合操作建议。

支持**日线**和**分钟级分时（MACDFS）**双周期，可多标的串行轮询。

## 安装

```bash
pip install akshare pandas numpy pyyaml
```

## 快速开始

```bash
# 日线监控默认标的（518880）
python main.py

# 多标的同时监控
python main.py -cs 518880,512880,510300

# 只执行一次检测
python main.py --once

# 5 分钟 MACDFS
# （先在 resources/config.yml 中将 period 改为 "5min"）
python main.py -cs 518880
```

## 命令行参数

| 参数 | 说明 |
| --- | --- |
| `-c, --code` | 单个 ETF 代码 |
| `-cs, --codes` | 多个 ETF 代码，逗号分隔，例如 `518880,512880` |
| `-i, --interval` | 轮询间隔（秒），**默认根据周期自动选择** |
| `-d, --history-days` | 历史 K 线天数 |
| `--once` | 只执行一次后退出 |

## 配置说明

所有配置通过 YAML 文件管理，无需改代码。

### `resources/config.yml` — 核心运行配置

```yaml
stock:
  code: "518880"          # 默认监控标的

monitor:
  poll_interval: 60       # 轮询间隔（秒），未指定时按周期自动选择
  history_days: 120       # 历史 K 线获取天数
  period: "daily"         # 数据周期: daily | 1min | 5min | 15min | 30min | 60min

indicators:
  macd:
    fast: 12
    slow: 26
    signal: 9
  kdj:
    n: 9
    m1: 3
    m2: 3
  volume:
    ma_days: 20
```

### `resources/strategy.yml` — 决策策略配置

```yaml
volume:
  ratio_threshold: 1.20   # 量比 > 1.2 视为显著放量
  ratio_shrink: 0.50      # 量比 < 0.5 视为显著缩量

macd_bar:
  significant_ratio: 1.50 # 柱体较均值放大 50% 视为显著
  recent_n: 8             # 参考近期 8 根柱体

divergence:
  lookback: 60            # 背离回溯 60 天/根
  peak_min_distance: 3    # 极值点最小间隔

score:
  strong_buy: 5           # 强烈看多阈值
  buy: 3                  # 偏多阈值
  sell: -3                # 偏空阈值
  strong_sell: -5         # 强烈看空阈值
```

参数已按 `resources/strategy_analysis_report.md` 量化分析报告优化，信号质量与实战可用性较初版有显著提升。

## 主要功能

- **MACD**：金叉/死叉、显著红绿柱（柱体动能放大检测）、顶/底背离
- **KDJ**：金叉/死叉、超买(J>100)/超卖(J<0)、背离
- **成交量**：放量/缩量、量价配合判断（量比 + 涨跌幅耦合）
- **综合评分**：多指标加权，按阈值输出"强烈看多 / 偏多 / 观望 / 偏空 / 强烈看空"建议
- **强烈信号强调**：当检测到显著柱体、背离、强 KDJ 信号或评分达强烈级别时，日志中会用 `⚠ 强烈信号` 区块醒目标识
- **自动周期适配**：切换 `config.yml` 中的 `period` 即可在日线与分钟级（1/5/15/30/60min）之间切换，数据源与轮询间隔自动适配
- **ETF 名称自动识别**：从 AKShare 接口实时获取，无需手动指定

├── logs/
│   └── monitor.log                  # 统一日志文件
